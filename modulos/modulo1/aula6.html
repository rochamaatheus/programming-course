Introdução às Funções

As funções são blocos fundamentais em JavaScript. Elas permitem encapsular lógica, facilitando a reutilização de código e a organização de programas. Vamos explorar como criar e utilizar funções de forma eficiente.

Criando Funções Básicas

Uma função é definida usando a palavra-chave function, seguida por um nome, parênteses e chaves. Dentro das chaves, colocamos o código que será executado quando a função for chamada.

Exemplo de Função Simples


function areaQuadrado(lado) {
  return lado * lado;
}

console.log(areaQuadrado(4)); // 16
console.log(areaQuadrado(5)); // 25
console.log(areaQuadrado(2)); // 4

No exemplo acima, definimos a função areaQuadrado que calcula a área de um quadrado com base no comprimento do lado fornecido. Ao chamar a função com diferentes valores, obtemos resultados distintos.

Função sem Parâmetros

Também podemos criar funções que não recebem parâmetros e retornam um valor fixo ou realizam uma ação específica. 


function pi() {
  return 3.14;
}

var total = 5 * pi(); // 15.700000000000001
console.log(total);

Nesse caso, a função pi retorna o valor de pi, que é então utilizado em um cálculo.

Parâmetros e Argumentos

Ao definir uma função, os valores dentro dos parênteses são chamados de parâmetros. Ao chamar a função, os valores passados são os argumentos.

Exemplo com Parâmetros


// 'peso' e 'altura' são os parâmetros
function imc(peso, altura) {
  const imc = peso / (altura ** 2);
  return imc;
}

// 80 e 1.80 são os argumentos
console.log(imc(80, 1.80)); // 24.691358024691358
console.log(imc(60, 1.70)); // 20.761245674740486

Podemos definir quantos parâmetros forem necessários, separados por vírgulas. Lembre-se de que os argumentos devem ser passados na mesma ordem dos parâmetros.

Executando Funções

Para executar uma função, utilizamos o nome da função seguido de parênteses. Se a função requer parâmetros, devemos fornecê-los dentro dos parênteses.

Exemplo de Chamada de Função


function corFavorita(cor) {
  if (cor === 'azul') {
    return 'Você gosta do céu';
  } else if (cor === 'verde') {
    return 'Você gosta de mato';
  } else {
    return 'Você não gosta de nada';
  }
}

console.log(corFavorita('azul')); // 'Você gosta do céu'
console.log(corFavorita()); // 'Você não gosta de nada'

Se não chamarmos a função, nada dentro dela será executado. É importante sempre lembrar de executar a função quando necessário.

Funções como Argumentos

Em JavaScript, podemos passar funções como argumentos para outras funções. Essas são chamadas de callbacks. São muito utilizadas em eventos e operações assíncronas.

Exemplo com Callback


addEventListener('click', function() {
  console.log('Clicou');
});
// A função possui dois argumentos:
// 1º é a string 'click'
// 2º é uma função anônima que será executada quando o evento ocorrer

Funções anônimas são aquelas sem um nome específico e são definidas no momento em que são passadas como argumentos.

Retorno de Funções

Uma função pode ou não retornar um valor. Se não definirmos um return, a função retornará undefined. O código dentro da função será executado normalmente.

Exemplo sem Retorno


function imc(peso, altura) {
  const imc = peso / (altura ** 2);
  console.log(imc);
}

imc(80, 1.80); // Exibe o IMC no console
console.log(imc(80, 1.80)); // Exibe o IMC e 'undefined'

No exemplo acima, a função imc não retorna um valor, apenas exibe o resultado no console.

Tipos de Retorno

Uma função pode retornar qualquer tipo de dado, incluindo outras funções. Porém, é importante manter a consistência nos tipos de retorno para evitar confusões.

Exemplo de Retorno Condicional


function terceiraIdade(idade) {
  if (typeof idade !== 'number') {
    return 'Informe a sua idade!';
  } else if (idade >= 60) {
    return true;
  } else {
    return false;
  }
}

console.log(terceiraIdade(65)); // true
console.log(terceiraIdade(45)); // false
console.log(terceiraIdade('sessenta')); // 'Informe a sua idade!'

Neste exemplo, a função retorna valores de tipos diferentes (string, boolean). Embora funcione, é uma boa prática manter o tipo de retorno consistente.

Escopo de Funções

Variáveis definidas dentro de uma função não são acessíveis fora dela. Isso é chamado de escopo. Isso ajuda a evitar conflitos entre variáveis com o mesmo nome em diferentes partes do código.

Exemplo de Escopo


function precisoVisitar(paisesVisitados) {
  var totalPaises = 193;
  return `Ainda faltam ${totalPaises - paisesVisitados} países para visitar`;
}

console.log(precisoVisitar(20)); // 'Ainda faltam 173 países para visitar'
console.log(totalPaises); // Erro: totalPaises is not defined

A variável totalPaises está definida dentro da função e não é acessível fora dela.

Escopo Léxico

Funções em JavaScript possuem acesso ao escopo em que foram criadas. Isso significa que elas podem acessar variáveis definidas no escopo pai.

Exemplo de Escopo Léxico


var profissao = 'Programador';

function dados() {
  var nome = 'Matheus';
  var idade = 18;

  function outrosDados() {
    var endereco = 'Santa Catarina';
    var idade = 29;
    return `${nome}, ${idade}, ${endereco}, ${profissao}`;
  }

  return outrosDados();
}

console.log(dados()); // 'Matheus, 29, Santa Catarina, Programador'
console.log(outrosDados()); // Erro: outrosDados is not defined

A função outrosDados acessa variáveis do escopo pai (nome, profissao). Porém, não é acessível fora da função dados.

Hoisting em Funções

Em JavaScript, funções declaradas usando function são "elevadas" ao topo do escopo, permitindo que sejam chamadas antes de serem definidas no código. Isso é conhecido como hoisting.

Exemplo de Hoisting


imc(80, 1.80); // Executa corretamente

function imc(peso, altura) {
  const imc = peso / (altura ** 2);
  console.log(imc);
}

Mesmo chamando a função antes de sua declaração, o código funciona devido ao hoisting. Porém, é recomendado declarar funções antes de utilizá-las para melhorar a legibilidade.
